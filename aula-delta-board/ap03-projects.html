Introdução
03. Conceito — Os 4 elementos de um Project
Da conversa que recomeça do zero ao assistente fixo com memória

Uma conversa normal no Claude começa do zero toda vez. Você precisa reexplicar quem você é, onde trabalha, o que faz. Um Project resolve isso de forma permanente: você configura o contexto uma vez, e o Claude carrega tudo automaticamente em cada nova conversa dentro daquele espaço.

Analogia

É a diferença entre contratar um novo freelancer toda semana e ter um assistente fixo que já conhece a sua empresa, o seu setor e as suas prioridades.

Para criar um Project, você preenche 4 elementos:

1
Nome do Project
Como esse assistente vai aparecer na sua lista (ex: "Assistente Executivo — João").
2
Descrição do Project
Texto curto explicando o objetivo do Project (visível no card do Project).
3
Instruções (System Prompt)
Texto que define a identidade, o contexto e o comportamento do assistente.
4
Arquivos de contexto
Documentos (.md, PDF, .docx, etc.) que o assistente lê em toda nova conversa dentro do Project.
💡 Por que esta atividade é diferente
Em vez de redigitar tudo, deixe o Claude gerar os 4 elementos prontos

O Claude já conhece você pelas APs 01 e 02 — cargo, segmento, mapa Kai-Fu Lee e inteligência competitiva. Nesta atividade você vai pedir a ele para entregar os 4 elementos prontos para colar/upload no Project que você vai criar. Sem retrabalho, com consistência total.

Demonstração
04. O que você vai ver
Fluxo completo em duas fases — geração no chat antigo e configuração do Project
Aviso

A demonstração a seguir é apenas para ilustração durante a aula expositiva. Você não precisa testar agora — você terá sua própria atividade prática logo depois desta seção.

O professor demonstra o fluxo completo em duas fases:

Fase 01 — Geração no chat das APs 01 e 02

O professor volta ao chat acumulado e pede ao Claude que gere os 4 elementos do Project de uma vez:

  • 3 textos simples (Nome, Descrição, Instruções)
  • 2 arquivos .md (os arquivos de contexto resumindo AP01 e AP02)

Em cerca de 1 minuto, o Claude entrega tudo pronto para uso.

Fase 02 — Configuração do Project

O professor cria um Project novo no Claude e preenche cada um dos 4 campos com o conteúdo gerado na Fase 01. Em seguida, testa com uma pergunta de validação. A resposta do Project é completamente diferente da que viria de um chat genérico — ancorada no setor, no cargo e nos casos reais que o aluno pesquisou.

Demonstração
06. O que observar durante a demonstração
4 sinais que diferenciam um Project com contexto de um chat genérico
1
O Claude entrega os elementos 1, 2 e 3 como blocos de texto prontos para copiar e colar nos campos correspondentes do Project.
2
Os elementos 4A e 4B vêm como artefatos .md separados, com botão de download.
3
Os textos não são templates genéricos — vêm customizados com o contexto do aluno (nome, cargo, segmento).
4
A configuração do Project leva ~3 minutos: criar → preencher 4 campos → testar.
Demonstração
07. Pergunta de teste após a configuração
A validação final — só responde bem com o contexto carregado

Para validar que o Project está funcionando, o professor faz uma pergunta que só pode ser respondida bem se o assistente tiver o contexto carregado:

Pergunta de validação
Considerando minha rotina, meu setor e o que os concorrentes estão fazendo com AI, qual seria a primeira iniciativa que eu deveria executar nos próximos 30 dias?
O que acontece

Em um chat normal, essa pergunta receberia uma resposta genérica. No Project recém-configurado, recebe uma recomendação específica baseada nas Quick Wins da Matriz Kai-Fu Lee, no segmento do aluno e no que os concorrentes mapeados na AP02 estão fazendo.

Atividade Prática
08. Seu Assistente Executivo permanente
Sua vez — gerar os 4 elementos no chat antigo e configurar o Project

Você vai criar agora seu próprio Assistente Executivo de AI — um Claude Project configurado com todo o contexto profissional que você construiu nas APs 01 e 02. Esta atividade tem duas fases:

  • Fase 01 — gerar os 4 elementos do Project no chat antigo (Nome + Descrição + Instruções + 2 arquivos .md)
  • Fase 02 — criar o Project, preencher os 4 campos com o conteúdo gerado, e testar
Duração total
12 min
estimado
Modelo recomendado
Fase 01Claude Opus 4.7 — geração
Fase 02Claude Sonnet 4.6 — uso do Project
Na Fase 01 você continua em Opus 4.7 (mesmo modelo das APs 01 e 02). No Project que você vai criar na Fase 02, recomendamos Sonnet 4.6 — é mais rápido e mais barato para uso diário, ideal para um assistente que você vai consultar várias vezes ao dia depois do curso.
💡 Por que duas fases
Em vez de redigitar tudo, aproveite a janela de contexto que você construiu

O Claude já conhece todo seu contexto profissional pelas APs 01 e 02. Vamos pedir a ele para gerar os 4 elementos prontos — isso elimina retrabalho, garante consistência e mostra na prática como uma janela de contexto rica vira um Assistente Executivo customizado em poucos minutos.

Passo 03 — Cole o prompt no Claude e envie

Cole o prompt no campo de mensagem do chat antigo (Cmd+V no Mac ou Ctrl+V no Windows) e clique em enviar. O Claude vai gerar os 4 elementos em sequência: 3 textos simples + 2 artefatos .md.

Passo 04 — Baixe os 2 arquivos .md E4A E4B

Localize os dois artefatos .md na resposta do Claude (mapeamento-ai-kaifulee.md e inteligencia-competitiva.md) e clique no botão Baixar em cada um. Os 2 arquivos serão salvos no seu computador.

FASE 02 Configuração
Crie o Project e preencha os 4 campos no Claude
Passo 05 — Acesse a área de Projects no Claude

Abra um chat novo do Claude (em claude.ai/new) e clique no ícone do menu lateral. No menu, clique em Projetos (ou Projects dependendo do idioma da interface).

Passo 06 — Crie um novo Project

Clique no botão + Novo projeto (ou + New project).

Passo 07 — Cole o Nome do Project E1 (Campo 1)

Volte ao chat antigo e copie o Elemento 1 — Nome do Project gerado pelo Claude. Clique no campo "No que você está trabalhando?" do formulário do Project e cole o conteúdo (Cmd+V no Mac ou Ctrl+V no Windows).

Passo 08 — Cole a Descrição do Project E2 (Campo 2)

Volte ao chat antigo e clique no ícone de copiar do bloco do Elemento 2 — Descrição do Project. Cole no campo "O que você quer realizar?" do formulário. Em seguida, clique em Criar projeto.

Passo 09 — Abra as instruções do Project (Campo 3)

No Project recém-criado, localize a seção Instruções e clique em Editar Instruções.

Passo 10 — Cole as Instruções do Project E3 (Campo 3)

Volte ao chat antigo e clique no ícone de copiar do bloco do Elemento 3 — Instruções do Project (o System Prompt completo). Cole no campo de instruções do Project. Em seguida, clique em Salvar instruções.

Passo 11 — Adicione os 2 arquivos de contexto E4A E4B (Campo 4)

Na seção Arquivos do Project, clique no ícone + e selecione Carregar do aparelho. Faça upload dos 2 arquivos .md que você baixou no Passo 04:

  • mapeamento-ai-kaifulee.md
  • inteligencia-competitiva.md
Passo 12 — Teste o Project com a pergunta de validação

Abra um novo chat dentro do Project (não num chat genérico do Claude) e use o prompt abaixo. Antes de enviar, mude o modelo para Sonnet 4.6 — mais rápido e barato para uso diário do seu assistente.

Prompt de teste — valida o Project
Considerando minha rotina, meu setor e o que os concorrentes estão fazendo com AI, qual seria a primeira iniciativa que eu deveria executar nos próximos 30 dias?
Prompt copiado. Cole no chat dentro do Project recém-criado.
