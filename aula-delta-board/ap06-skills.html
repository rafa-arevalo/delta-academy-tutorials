Introdução
01. Objetivo de aprendizagem
Uma Skill, um atalho, um diagnóstico pronto

Entender o que é uma Skill no Claude, criar a sua primeira Skill — de análise de OKRs — e usá-la para gerar um diagnóstico executivo do seu departamento.

Na próxima atividade, essa Skill vai alimentar a criação de uma apresentação PPT para o board.

Introdução
02. Conceito — Skills
Transforme um prompt recorrente em um atalho permanente

Um prompt comum você escreve uma vez e descarta. Uma Skill é diferente: é um prompt que você salva com nome, contexto e instruções fixas — e aciona com um comando curto sempre que precisar.

É a diferença entre digitar o mesmo pedido toda semana e ter um atalho que faz isso em segundos.

Skills fazem mais sentido para tarefas recorrentes com output padronizado. Análise de OKRs é um exemplo perfeito: todo mês ou trimestre você tem novos dados, mas a análise que você quer sempre segue o mesmo padrão — diagnóstico por objetivo, Key Results em risco, recomendações de ação.

Lógica

Você cria a Skill uma vez. Depois é só colar os dados e acionar.

Skill de análise de OKRs criada ao vivo no contexto de logística

Criação ao vivo de uma Skill de análise de OKRs para o contexto de logística. O professor mostra a estrutura de uma Skill — o que cada parte faz — e depois aciona a Skill com uma planilha fictícia de OKRs.

O Claude devolve um diagnóstico executivo em segundos.

Demonstração
04. O que observar
4 características de uma Skill bem feita
  • Três partes. Uma Skill tem nome (como você a aciona), instrução (o que o Claude deve fazer) e contexto fixo (o que o Claude já sabe antes de você enviar os dados).
  • O poder está no contexto fixo. Você não precisa reexplicar o que é um OKR, qual é o seu cargo ou o que espera de formato — o Claude já sabe.
  • Reutilização total. A mesma Skill roda sobre qualquer planilha de OKRs — você só troca os dados, o restante é automático.
  • Output padronizado. O diagnóstico chega sempre no mesmo formato, mesmo nível de profundidade, pronto para levar para uma reunião.
Demonstração
05. Prompt usado para criar a Skill na demonstração
A estrutura completa: nome, instrução, regras de formato
Prompt da demo — criação da Skill /analise-de-okr
Crie uma Skill chamada "/analise-de-okr" com as seguintes instruções: NOME DA SKILL: /analise-de-okr INSTRUÇÃO PARA O CLAUDE: Você é um consultor executivo especialista em gestão por objetivos (OKR). Quando eu acionar esta Skill e enviar uma planilha de OKRs, analise os dados e entregue: 1. DIAGNÓSTICO GERAL (1 parágrafo) Avaliação do desempenho geral do trimestre — o que está indo bem, o que está em risco e qual é o padrão dominante nos resultados. 2. ANÁLISE POR OBJETIVO Para cada objetivo, entregue: - Status geral (No prazo / Em risco / Atrasado) - Key Result com melhor desempenho - Key Result mais crítico - 1 recomendação de ação para o próximo período 3. RANKING DE PRIORIDADE Liste os 3 Key Results que precisam de atenção imediata, em ordem de criticidade, com justificativa em 1 linha cada. 4. PRÓXIMOS PASSOS 3 ações concretas para o gestor tomar na próxima semana com base nos dados. REGRAS DE FORMATO: - Linguagem executiva e direta — sem jargão acadêmico - Máximo 1 página - Use os dados exatamente como enviados — não invente métricas ou resultados - Se algum dado estiver ausente ou inconsistente, sinalize antes de analisar

Atividade Prática
08. Passo a passo
4 passos em até 9 minutos — tudo dentro do seu Project
1
Gere a sua planilha de OKRs. (2 min)
Abra o seu Project e cole o Prompt A (abaixo). O Claude vai gerar uma planilha fictícia com OKRs realistas para o seu cargo e setor. Copie a tabela gerada.
2
Crie a Skill de análise de OKRs. (3 min)
No chat do seu Project, cole o Prompt B (abaixo) para criar a Skill.
3
Acione a Skill com a sua planilha. (2 min)
No chat do Project, digite /analise-de-okr e cole a tabela do Passo 1 logo em seguida (ver Prompt C).
4
Leia o diagnóstico. (2 min)
Observe: vocabulário do seu setor, KRs em risco identificados, recomendações específicas — não genéricas. O formato é o mesmo que você definiu na Skill, independentemente dos dados.
Prompt A — gerar sua planilha de OKRs
Com base no meu contexto profissional salvo neste Project, crie uma planilha de OKRs fictícia mas realista para o meu departamento referente ao último trimestre. A planilha deve ter estas colunas: Objetivo, Key Result, Meta, Resultado Atual, % de Atingimento, Status Inclua: - 3 objetivos estratégicos típicos do meu cargo e setor - 3 Key Results por objetivo (total de 9 KRs) - Resultados variados: alguns acima da meta, alguns abaixo, alguns em risco — para a análise ser interessante Entregue em formato de tabela pronta para copiar.
