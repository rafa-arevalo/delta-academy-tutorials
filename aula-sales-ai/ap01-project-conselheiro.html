Introdução
02. Os 4 elementos do Project
Os blocos que você preenche para criar o Project Conselheiro

Para criar o Project Conselheiro, você preenche 4 elementos. Nesta AP01 vamos configurar os 3 primeiros — o 4º é opcional e será preenchido na AP02 com a Matriz Kai-Fu Lee.

Elemento 1
Nome do Project
Como o ambiente vai aparecer na sua lista de Projects. Ex.: "Meu Conselheiro de IA — Gerente Comercial".
Elemento 2
Descrição do Project
Texto curto explicando o objetivo do Project — seu conselheiro pessoal de IA para decisões do dia a dia.
Elemento 3
Instruções (System Prompt)
Texto que define como o conselheiro deve se comportar: identidade, tom, contexto sobre você (cargo, segmento, responsabilidades, equipe) e diretrizes de resposta.
Elemento 4 · opcional
Arquivos de contexto
Documentos que o Project consulta em toda conversa — a biblioteca particular do seu conselheiro. Preenchido na AP02 com a Matriz Kai-Fu Lee.
Dica

Para que serve a Knowledge Base do Project? Pense nela como a biblioteca particular do seu conselheiro. Sempre que você fizer uma pergunta, o Claude consulta esses documentos antes de responder — sem você precisar anexar nada. Exemplos do que cabe ali: plano de contas da empresa, política comercial, manual de vendas, relatórios trimestrais, OKRs do time, atas das últimas reuniões de board. Nesta aula, o primeiro arquivo de contexto vai surgir naturalmente na AP02: o HTML da Matriz Kai-Fu Lee gerado pelo próprio Claude — você só vai precisar baixar e subir como arquivo de contexto. Por isso a AP01 não precisa de arquivos prévios.

03. A jornada da aula
2 fases sequenciais para sair de mãos vazias e terminar com o conselheiro pronto

Esta atividade prática tem 2 fases sequenciais. Você sai de mãos vazias e termina com um conselheiro de IA pronto pra responder.

Fluxo da AP01

  • FASE 01 — Geração no chat (1 mensagem). Você abre um chat novo no Claude, usa o gerador interativo de prompt para personalizar com suas variáveis (segmento, cargo, responsabilidades, equipe) e cola o prompt. O Claude devolve em uma única resposta os 3 textos prontos (Nome, Descrição, Instruções) + 2 perguntas copiáveis para você testar o conselheiro.
  • FASE 02 — Configuração do Project (1 mensagem). Você cria um Project novo no Claude e cola cada texto no campo correspondente. Ao final, cola uma das 2 perguntas dentro do Project pra ver o conselheiro funcionando.
Introdução
04. Conceito — Chat comum vs Project
A diferença entre começar do zero e ter contexto permanente
Chat comum
Consultor novo a cada reunião
Quando você abre um chat novo no Claude, toda conversa começa do zero. Você precisa explicar quem é, qual seu cargo, em que setor atua, quais decisões está tomando. A cada nova conversa, mesma explicação. O Claude é brilhante, mas é um consultor que acabou de te conhecer.
Project
Conselheiro fixo que já te conhece
Um Project resolve isso de forma permanente: você configura quem você é e como o assistente deve se comportar uma única vez, e o Claude carrega tudo automaticamente em toda nova conversa dentro do Project. Toda pergunta já parte com o contexto carregado.
Dica

É a diferença entre contratar um consultor novo a cada reunião e ter um conselheiro fixo que já conhece sua empresa, sua equipe, suas métricas e seus desafios. O Project é o ambiente onde esse conselheiro mora.

O professor demonstra o fluxo completo de criação do Project Conselheiro em duas fases — usando uma persona fictícia para ilustrar (Gerente Comercial em empresa de SaaS B2B):

Fase 01 — Geração dos elementos no chat (1 mensagem)

O professor abre um chat novo no Claude (modelo Sonnet 4.6) e cola um prompt pré-preenchido com a persona-exemplo. Em uma única mensagem, o Claude devolve:

  • Nome do Project pronto (copiável)
  • Descrição pronta (copiável)
  • Instruções (System Prompt completo) prontas (copiáveis)
  • 2 perguntas iniciais para testar o conselheiro (cada uma copiável individualmente)

Fase 02 — Configuração do Project (1 mensagem)

O professor cria um Project novo no Claude, preenche os 3 campos com o conteúdo gerado e testa o conselheiro colando uma das 2 perguntas — mostrando como a resposta vem totalmente contextualizada (ao contrário de um chat comum).

Dica

Pedir todos os elementos do Project num único prompt em vez de várias perguntas separadas é uma técnica de engenharia de prompt eficiente que economiza limite no plano gratuito sem perder qualidade — e ainda garante consistência entre os textos gerados.

Demonstração
07. O que observar durante a demonstração
4 sinais que diferenciam um Project Conselheiro bem configurado de um chat genérico
  • 1. Blocos separados. O Claude entrega os 4 elementos como blocos de texto separados, prontos para copiar e colar nos campos correspondentes do Project — sem mistura entre Nome, Descrição e Instruções. As 2 perguntas do Elemento 4 também vêm em blocos separados.
  • 2. Não é template genérico. O System Prompt vem customizado com o contexto profissional da persona (SaaS B2B, Gerente Comercial, time de 8 vendedores, frameworks MEDDIC/SPIN).
  • 3. Resposta sem pedir contexto. Quando o professor cola uma das 2 perguntas dentro do Project, a resposta não pede contexto de volta — vai direto à recomendação, porque o contexto já está carregado nas Instruções.
  • 4. Tudo numa única resposta. Todo o material da AP aparece numa única resposta do Claude — confirmando que uma só mensagem foi suficiente.

08. Sua atividade prática
Agora é a sua vez — criar o seu Project Conselheiro personalizado

Você vai criar agora o seu Project Conselheiro de IA personalizado — o ambiente que vai abrigar o conselheiro estratégico que conhece o seu cargo, segmento e responsabilidades. Esta AP tem duas fases e dura cerca de 25 minutos.

Duração
25
minutos
Modelo recomendado
Claude Sonnet 4.6
Modelo padrão do plano gratuito do Claude, acessível para todos os alunos. Se você tem acesso ao plano pago, pode usar o Opus 4.7 para System Prompts ainda mais sofisticados.
Dica

Em vez de redigitar manualmente um System Prompt completo do zero, você pede ao Claude para gerar tudo de uma vez no chat — personalizado com as suas variáveis profissionais. Isso elimina retrabalho, garante qualidade no System Prompt e demonstra na prática como um prompt bem estruturado economiza mensagens — algo essencial para quem usa o plano gratuito.

Atividade Prática
09. Passo a passo
8 passos — do chat novo até o conselheiro respondendo dentro do Project
Passo 01 — Abra um chat novo no Claude

Abra claude.ai/new no navegador e faça login se necessário. Confirme que o modelo selecionado é Claude Sonnet 4.6 — disponível tanto no plano gratuito quanto no pago.

Passo 02 — Personalize e copie o prompt do Conselheiro

Use o gerador interativo abaixo para preencher os 4 campos com suas variáveis profissionais. À medida que você preenche, a pré-visualização do prompt mostra seus dados em verde. Quando todos os campos estiverem preenchidos, o botão "Copiar prompt" é habilitado.

Gerador Prompt do Conselheiro de IA
Preencha os 4 campos — o prompt aparece pronto abaixo, com seus dados destacados em verde.
Preencha todos os campos antes de copiar. Copiado! Cole no chat do Claude que você acabou de abrir no Passo 01.
Pré-visualização do prompt
Passo 03 — Cole o prompt no Claude e envie

Volte para o chat do Claude que você abriu no Passo 01. Cole o prompt copiado (Cmd+V no Mac ou Ctrl+V no Windows) no campo de mensagem e clique no botão de envio. Tempo de resposta: 60–90 segundos.

Passo 04 — Confirme a resposta com os 4 elementos

O Claude vai entregar os 4 elementos numerados como blocos separados (Nome, Descrição, Instruções e 2 perguntas). Mantenha esta aba aberta — você vai voltar aqui várias vezes na Fase 02 para copiar cada elemento.

Passo 05 — Acesse a área de Projects no Claude

No menu lateral do Claude, clique em Projetos (ou Projects dependendo do idioma da interface). Em seguida, clique no botão + Novo projeto para abrir o formulário de criação.

Passo 06 — Cole o Nome e a Descrição do Project

Volte ao chat da Fase 01 e copie o Elemento 1 — Nome. Cole no campo "No que você está trabalhando?" do formulário. Em seguida, copie o Elemento 2 — Descrição e cole no campo "O que você quer realizar?". Clique em Criar projeto para finalizar a configuração inicial.

Passo 07 — Cole as Instruções do Project (System Prompt)

No Project recém-criado, localize a seção Instruções e clique no ícone para adicionar/editar. Volte ao chat da Fase 01, copie o Elemento 3 — Instruções (todo o System Prompt) e cole no editor. Em seguida, clique em Salvar instruções.

Importante

O System Prompt do Elemento 3 é longo (vários parágrafos). Confirme que copiou o bloco inteiro — do "Identidade:" até a última linha das "Diretrizes de resposta". Se cortar pela metade, o conselheiro perde contexto.

Passo 08 — Teste o conselheiro com uma das 2 perguntas

Dentro do Project, abra uma nova conversa. Volte ao chat da Fase 01, copie a Pergunta 1 do Elemento 4 e cole no campo de mensagem do Project. Envie. Validação visual: a resposta do Claude vem direto ao ponto, sem pedir contexto de volta — ele já sabe quem você é, qual seu cargo e suas responsabilidades.

Desafio
11. Implementando na sua empresa
3 ações para a próxima semana

Agora que você tem o conselheiro funcionando, o desafio é integrar ele de verdade na sua rotina — não deixar ele esquecido depois da aula.

1
Use o conselheiro em pelo menos 3 decisões reais desta semana. (15–30 min total)
Pode ser qualquer coisa: como abordar um cliente, como conduzir uma reunião difícil, como estruturar uma proposta. O importante é levar problemas reais, não exercícios.
2
Refine as Instruções pelo menos uma vez.
Depois de 3 conversas, você vai perceber padrões — talvez o conselheiro responda longo demais, ou pouco provocativo, ou que falta um framework específico do seu setor. Edite as Instruções do Project e ajuste.
3
Compartilhe um print de uma resposta do conselheiro que te ajudou genuinamente — com um colega da sua empresa ou no grupo da turma. Isso reforça o aprendizado e abre espaço pra discussão sobre como cada um está usando.

Tempo estimado

15 a 30 minutos espalhados ao longo da semana — uso real do conselheiro nas decisões do dia a dia.

Insight

Um Project só fica bom com uso real. As Instruções que você configurou hoje são a versão 0.1 do seu conselheiro — a versão 1.0 vem depois de 3 a 5 conversas reais, quando você ajusta tom, profundidade e formato com base no que funciona pra você. Trate o conselheiro como uma relação que evolui, não como uma ferramenta estática.
