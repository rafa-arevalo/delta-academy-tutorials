00. Introducao
O que voces vao aprender neste modulo

A qualidade da resposta do Claude depende diretamente da qualidade do pedido que voce faz. Um prompt ruim entrega uma resposta generica que voce vai precisar editar por minutos. Um prompt bom entrega algo que voce usa de verdade — direto do Claude para o trabalho.

Neste modulo voces aprendem a estruturar pedidos que funcionam e a usar o Artifact — uma janela do Claude que transforma qualquer resposta em um documento, tabela, grafico ou codigo pronto para uso, sem precisar copiar e colar nada.

Objetivos de Aprendizagem

  1. Identificar a diferenca entre um prompt ruim e um prompt bom
  2. Aplicar os 4 elementos de um prompt bom em qualquer pedido
  3. Entender o que e um Artifact e como ativa-lo
  4. Gerar um documento profissional no padrao Bram Offshore como Artifact
Teoria
01. Prompt ruim vs. prompt bom
O mesmo pedido feito de duas formas diferentes

Veja o mesmo pedido feito de duas formas diferentes. O tema e identico. O resultado e completamente diferente.

Prompt ruim

Me ajuda com uma checklist de inspecao de embarcacao.
Resultado: texto corrido, generico, sem estrutura. Voce vai precisar de 10 minutos de edicao para transformar isso em algo utilizavel.

Prompt bom

Voce e tecnico de seguranca offshore certificado pela Marinha do Brasil. Crie uma checklist de inspecao pre-operacional para embarcacao de apoio maritimo, no formato de documento profissional como Artifact em HTML. A checklist deve cobrir: sistemas de navegacao, equipamentos de salvatagem, HSE, motores e documentacao regulatoria (NORMAM-01). Inclua checkboxes, campo para numero da embarcacao, data e assinatura do responsavel. Pronto para uso operacional.
Resultado: Artifact com documento completo, formatado e pronto para imprimir ou enviar.

Os 4 elementos de um prompt bom

Todo prompt bom tem quatro elementos. Quando um esta faltando, a resposta fica generica.

#ElementoO que defineExemplo
1PapelQuem o Claude deve ser para responder bem"Voce e tecnico de seguranca offshore..."
2ContextoOnde, quando e para quem"...embarcacao de apoio maritimo, operacao offshore brasileira..."
3Tarefa claraO que exatamente voce quer"...crie uma checklist de inspecao com estas secoes..."
4Formato de saidaComo o resultado deve chegar"...com checkboxes, campos e assinatura do responsavel."
Nota do instrutor

Pecam para a turma identificar esses 4 elementos no prompt bom da comparacao acima. E um exercicio de 1 minuto que fixa o conceito antes de seguir.

02. O que e um Artifact
A diferenca entre resposta normal e Artifact

Quando voce pede ao Claude para criar um documento, tabela, grafico ou codigo, ele pode entregar esse conteudo de duas formas diferentes. A diferenca muda tudo.

Resposta normal
O Claude responde com texto corrido dentro do chat. Voce le, seleciona, copia, abre outro programa, cola, formata. Sao varios passos manuais ate ter algo utilizavel.
Artifact
BRAM OFFSHORE
Checklist de Inspecao
Sistemas de navegacao
Equipamentos de salvatagem
Sistemas HSE
Janela lateral — pronto para usar

O que pode virar um Artifact

📄
Documento
Relatorio, checklist, e-mail, proposta, ata de reuniao.
Como pedir: "Crie um documento com..." / "Gere um relatorio em HTML com..."
📊
Tabela
Planilha, comparativo, ranking, matriz de prioridades.
Como pedir: "Gere uma tabela em HTML com..." / "Crie um comparativo entre..."
📈
Grafico
Linha, barras, pizza, dispersao — o que voces fizeram no Modulo 01.
Como pedir: "Crie um grafico de linhas em HTML com..."
💻
Codigo
Scripts Python, JavaScript, SQL, formulas. Para tarefas tecnicas.
Como pedir: "Escreva um script Python que..." / "Gere o codigo HTML para..."
Regra

Sempre que quiser um Artifact, especifique o formato no final do prompt: "...no formato de documento em HTML" / "...como Artifact" / "...pronto para usar diretamente".

04. Demonstracao 2 — Prompt bom ao vivo
Mesmo tema. Agora com os 4 elementos.

Mesmo tema. Agora com os 4 elementos. O resultado vai aparecer como Artifact — ja no padrao visual da Bram Offshore.

Atividade Pratica
05. Atividade Pratica
Chegou a hora de voces

Dinamica da atividade

Agora e a vez de voces. Vao ter 3 minutos para criar um documento real do proprio departamento usando um prompt bom. Depois, o instrutor vai sortear 3 colegas para apresentar o resultado — cada um tera ate 2 minutos.

O que fazer

1
Escolham um processo do departamento — algo recorrente que voces ja observaram, participaram ou que sabem que acontece no dia a dia. Pode ser simples.
2
Preencham o prompt abaixo com o contexto de voces — substituam os campos entre colchetes.
3
Rodem o prompt no Claude e aguardem o Artifact.
Exemplo — como um colega faria

Ana Beatriz, trainee de RH

Departamento: Recursos Humanos Funcao: Trainee de RH Processo escolhido: Onboarding de novo colaborador Etapas que conheco: envio de documentos, cadastro no sistema, apresentacao a equipe, entrega de equipamentos, assinatura de contrato

Resultado esperado: Um Artifact com uma checklist de onboarding estruturada em secoes (pre-chegada, primeiro dia, primeira semana), com checkboxes, campo para nome do colaborador, data de entrada e assinatura do responsavel de RH — tudo no padrao visual da Bram Offshore.

O resultado de cada um de voces vai ser completamente diferente — porque cada departamento tem processos diferentes.

Prompt guiado — Voces fazem agoraVoce e um profissional especializado em [seu departamento] na Bram Offshore, empresa brasileira de servicos de apoio maritimo offshore. Crie uma checklist operacional para o seguinte processo do meu departamento: [nome do processo — ex: "onboarding de novo colaborador" / "inspecao de equipamento" / "fechamento mensal" / "atendimento de chamado de TI" / "revisao de contrato"]. Etapas que ja conheco ou imagino que esse processo tem: [descreva de 3 a 5 etapas que voce ja sabe ou imagina] Gere o documento como Artifact em HTML com identidade visual BRAM OFFSHORE: - Header com fundo azul marinho (#003087), texto branco, "BRAM OFFSHORE" em caixa alta e nome do departamento abaixo - Titulos das secoes: azul marinho (#003087), Montserrat SemiBold - Checkboxes para cada item da checklist - Campos para: responsavel, data e assinatura - Rodape com fundo azul marinho (#003087) e texto branco - Estilo corporativo, limpo e profissional — pronto para uso real

O que preparar para apresentar

Se voce for sorteado, prepare para contar em ate 2 minutos:

  1. Qual processo escolheu — e por que ele e relevante para o seu departamento na Bram
  2. Mostre o Artifact gerado na tela — compartilhe o resultado com a turma
  3. Compare com o prompt ruim — o que teria acontecido se voce tivesse pedido apenas "me faz uma checklist de [processo]"?
  4. Qual dos 4 elementos voce considera mais importante no prompt que escreveu — e por que

Entregavel do modulo

  • Escolheu um processo real do proprio departamento
  • Aplicou os 4 elementos no prompt
  • O Artifact foi gerado no padrao visual da Bram Offshore
  • Consegue explicar a diferenca entre o prompt ruim e o prompt bom que escreveu
